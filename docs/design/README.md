# Emberline status bar icon — approved design

[**status-icon-design.html**](status-icon-design.html) is the approved design (open
in a browser or VS Code **Live Preview**). It is the visual source of truth for the
status bar icon shipped by the extension.

## The mark

`icon.png` as a monochrome glyph:

- **Lines 1 & 2** — hollow, border only.
- **Line 3** — hollow on the left, then vertical strokes grow denser toward the
  right until the end is **solid** (no gap) — lit by the cursor.
- **The cursor** — a fully-filled rounded **rectangle**, split off the end of line 3.

## States

| State | Glyph | Motion / colour |
|-------|-------|-----------------|
| **Ready** | line 3 lit + solid cursor | warm ember, static |
| **Thinking** | line 3 fill sweeps right→left; lines 1 & 2 stay dark | warm ember, animated (frame-swap) |
| **Starting** | all lines dark, cursor blinks | warm ember, animated (frame-swap) |
| **Disabled** | hollow lines + hollow cursor + slash | dimmed, static |
| **Error** | ready glyph | `statusBarItem.errorBackground` pill |

## How it ships (constraints)

A status-bar glyph is **one flat colour**; the warm ember tint is a themeable colour
(`emberline.statusIcon`, warm on dark / deeper on light). VS Code's only CSS
animation is `~spin`, so the sweep and blink are done by the extension swapping a
small set of pre-rendered glyph frames on a timer (see `status.ts`). The glyph font
is generated by `extension/scripts/build-icon-font.mjs`; `icon.png` itself is
unchanged.
